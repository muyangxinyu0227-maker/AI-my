# Claude Code Workflow 工作流

## 核心原则

- **合理分工**：让 Claude 处理重复性工作，架构设计由人把控
- **建立信任**：从简单任务开始，观察执行效果
- **清晰上下文**：通过 CLAUDE.md 提供项目背景
- **验证环节**：AI 生成的代码需要测试验证

## 日常开发流程

```
1. 了解需求 → 2. 分析代码 → 3. 制定计划
→ 4. 编写代码 → 5. 测试调试 → 6. 代码审查 → 7. 提交代码
```

## 常用命令

| 命令 | 功能 |
|------|------|
| @directory src/ | 了解项目结构 |
| @grep "关键词" | 查找相关代码 |
| /plan | 制定实施计划 |
| /review | 代码审查 |
| /commit | 生成提交信息 |

## 新功能开发 Checklist

| 阶段 | 检查项 |
|------|--------|
| 开始前 | 了解项目结构 |
| 编码中 | 制定实施计划 |
| 测试阶段 | 运行单元测试 |
| 审查阶段 | 自动代码审查 |
| 提交阶段 | 生成提交信息 |

## Bug 修复流程

1. 收集错误信息
2. 定位问题代码
3. 分析根因
4. 实施修复
5. 提交修复
