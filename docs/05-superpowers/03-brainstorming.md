# 需求澄清

## brainstorming 技能

当需求模糊时，Claude 会通过提问帮你澄清。

## 工作方式

```
你：做一个博客系统

Claude：
- 这个博客给谁看？
- 需要支持 Markdown 吗？
- 需要评论功能吗？
- 是单用户还是多作者？
```

## 为什么要澄清？

- 避免做无用功
- 确保理解需求
- 减少返工

## 使用场景

- 需求不明确时
- 功能描述模糊时
- 不确定具体细节时

## 触发方式

- 需求模糊时自动触发
- 使用 brainstorming 技能

## 练习

尝试输入模糊需求，观察 Claude 如何提问：

```
做一个用户系统
```
