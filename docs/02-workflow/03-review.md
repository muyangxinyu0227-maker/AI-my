# 代码审查

## 审查流程

```
1. 提交前自审 → 2. 发起审查 → 3. 修复问题 → 4. 合并代码
```

## 审查维度

| 优先级 | 维度 | 说明 |
|--------|------|------|
| 🔴 高 | 安全性 | SQL 注入、密码硬编码 |
| 🟡 中 | 性能 | N+1 查询、内存泄漏 |
| 🟢 低 | 代码质量 | 命名规范、重复代码 |

## 使用 /review

```text
/review
```

或指定文件：

```
审查 src/utils/auth.ts 文件
```

## 审查内容

### 1. 安全性

- 敏感信息处理
- 权限控制
- 输入验证

### 2. 性能

- 数据库查询效率
- 内存使用
- 算法复杂度

### 3. 代码质量

- 命名规范
- 代码重复
- 注释完整性

### 4. 测试覆盖

- 单元测试
- 边界情况

## PR 描述模板

```markdown
## 做了什么
- 添加用户登录功能

## 为什么改
- 需要支持 JWT 认证

## 测试情况
- [x] 单元测试通过
- [x] 手动验证通过
```

## 最佳实践

1. 提交前先自审
2. 关注重点问题
3. 及时响应反馈
