# 开发流程

## 标准开发流程

```
1. 理解需求 → 2. 分析代码 → 3. 制定计划
→ 4. 编写代码 → 5. 测试调试 → 6. 代码审查 → 7. 提交代码
```

## 步骤详解

### 1. 理解需求

```text
我想添加一个用户登录功能，需要：
- 邮箱密码登录
- JWT token
- 记住我功能
```

### 2. 分析代码

```text
@directory src/
```

- 了解项目结构
- 查找相关代码
- 理解现有架构

### 3. 制定计划

```
使用 /plan 命令
```

或描述：

```
帮我制定一个实现计划
```

### 4. 编写代码

按计划逐步实现：
1. 先实现数据模型
2. 再实现核心逻辑
3. 最后处理边界情况

### 5. 测试调试

```text
!npm test
```

- 运行单元测试
- 手动测试功能
- 修复发现的问题

### 6. 代码审查

```
使用 /review
```

### 7. 提交代码

```
使用 /commit
```

## 常见陷阱

❌ 一次性给太多任务
✅ 分步骤完成任务

❌ 不提供足够的上下文
✅ 说明具体文件和需求

❌ 不验证 AI 生成的代码
✅ 每次修改后运行测试

## 最佳实践

1. 从简单任务开始
2. 清晰描述需求
3. 及时验证结果
4. 保持对话简洁
