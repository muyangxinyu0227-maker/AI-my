# Bug 修复

## Bug 修复流程

```
1. 收集信息 → 2. 定位问题 → 3. 分析根因
→ 4. 实施修复 → 5. 验证修复 → 6. 提交代码
```

## 步骤详解

### 1. 收集信息

提供以下信息：
- 错误消息
- 复现步骤
- 环境信息

示例：

```
登录功能报错 'Invalid token'，错误在 src/middleware/auth.ts
```

### 2. 定位问题

```text
@grep "Invalid token"
```

- 搜索相关代码
- 查看调用链
- 理解问题位置

### 3. 分析根因

- 理解为什么出错
- 评估影响范围
- 确定修复方案

### 4. 实施修复

- 编写修复代码
- 添加回归测试

### 5. 验证修复

```text
!npm test
```

- 运行相关测试
- 手动验证功能
- 确认问题已解决

### 6. 提交代码

```
/commit
```

## 常用命令

| 命令 | 用途 |
|------|------|
| @grep "错误信息" | 搜索错误位置 |
| @grep "functionName" -C 5 | 查看调用上下文 |
| !npm test | 运行测试 |
| !git diff | 查看改动 |

## 调试技巧

1. 优先提供错误信息
2. 说明复现步骤
3. 提供相关代码上下文
